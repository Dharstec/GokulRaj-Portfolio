Introduction

This project aims to develop a program that enables an RB1 robot to approach, detect, and attach to a warehouse shelf. The project is divided into two main tasks: creating a pre-approach motion node and developing a final approach motion node. Utilizing ROS2 for robot control and ROS1 for the simulation environment, the project demonstrates seamless integration and functionality in a simulated warehouse setting.

Objectives
  1. Pre-Approach Motion: Develop a ROS2 node to move the robot near the shelf.
  2. Final Approach Motion: Create a ROS2 node to detect the exact position of the shelf, maneuver the robot beneath it, and attach the robot to the shelf.
Tools and Technologies
  • Programming Languages: C++
  • Frameworks and Libraries: ROS2, ROS1
  • Simulation Environment: Gazebo
  • Version Control: Git

Process and Development
Task 1: Pre-Approach Motion

The first task involved setting up a ROS2 node named pre_approach.cpp, designed to subscribe to the /scan topic and publish to /robot/cmd_vel. This node was developed to move the robot forward until it detected an obstacle at a specified distance, at which point it would stop and rotate the robot to face the shelf. To facilitate this, a launch file pre_approach.launch.xml was created to start the node and set parameters for obstacle distance and rotation degrees. The outcome of this task was that the robot ended up directly in front of the shelf, facing it.[Image]

Task 2: Final Approach Motion

The second task involved developing a service server node named approach_service_server.cpp to handle the final approach. This node's functionality included detecting shelf legs using laser intensity values, publishing a transform (cart_frame) to the center point between the legs, moving the robot to these coordinates, and then moving it forward 30 cm to position it beneath the shelf. Once the robot was correctly positioned, it would lift the shelf. To integrate this process, pre_approach_v2.cpp was updated to call the /approach_shelf service after completing the pre-approach motion. A launch file attach_to_shelf.launch.py was created to start both the service server and the updated pre-approach node. The outcome of this task was that the robot successfully attached to the shelf by following the transforms and laser detections.[Video]

Results

The project demonstrates the ability to autonomously approach and attach to a shelf in a simulated warehouse environment. It highlights the effective use of ROS2 for control and ROS1 for simulation, showcasing the potential for complex robotic behaviors in industrial applications.

Key Insights
  • Modular Design: Using separate nodes and services for different tasks ensures modularity and ease of maintenance.
  • TF Integration: Accurate localization and tracking using TF frames are crucial for precise robot movements and interactions.
  • Laser Intensities: Leveraging laser intensity values aids in detecting specific objects (shelf legs) within the environment, enhancing the robot's operational accuracy.

This project provides a comprehensive example of integrating various ROS tools and techniques to achieve a complex robotic task, laying the groundwork for future enhancements and real-world applications.
