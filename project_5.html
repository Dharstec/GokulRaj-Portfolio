Robot chase game: Simulating Multi-Robot Pursuit Using ROS2 with TF Integration

Simulating playful interactions between autonomous robots in a controlled environment.

GIT LINK 1

Introduction
Skip To Content

This project aims to simulate a playful chase game between two autonomous robots named Rick and Morty. Using ROS2, Gazebo, and Rviz, the project demonstrates how robots can interact, track, and follow each other using TF frames and URDF/XACRO modeling.

×
Objectives

To develop a simulation in which one robot (Rick) can autonomously chase another robot (Morty) within a Gazebo environment. The project involves:

  • Modeling robots using URDF and refactoring with XACRO.
  • Simulating the robots in Gazebo and Rviz.
  • Implementing a ROS2 node in C++ for robot tracking and pursuit behavior.
Tools and Techniques:
  • Programming Languages: C++
  • Simulation Environment: Gazebo
  • Visualization Tool: Rviz
  • Robot Modeling: URDF (Unified Robot Description Format), XACRO
  • Middleware: ROS2 (Robot Operating System 2)
  • Version Control: Git

Process and Development
Initial Setup:

The initial setup involves cloning necessary repositories and setting up the ROS2 workspace. After downloading the required files, the workspace is compiled, and the Barista robot simulation is launched to ensure everything is correctly set up.

Simulation of Multiple Robots:

Launching Multiple Robots: A new launch file, barista_two_robots.launch.py, is introduced to initialize Gazebo and Rviz with both robot models simultaneously, ensuring unique namespaces to avoid any conflicts in node names, topics, and TF frames.

Validation: The setup is validated by confirming that both robots are displayed correctly and can be controlled independently using remapped command velocity topics.

Robot Chase:

Creating ROS2 Package: A new ROS2 package called robot_chase is initialized, with a source file named robot_chase.cpp added to implement the pursuit algorithm.

Implementing Pursuit Algorithm: A ROS2 node written in C++ listens to the TF frames of Morty and computes the distance and angular errors between Rick and Morty. Based on these errors, velocity commands are sent to Rick, enabling it to follow Morty.

Testing and Validation: The behavior of the pursuit algorithm is tested by moving Morty and verifying that Rick correctly chases Morty, adjusting its movements to follow and try to catch Morty.

Results

The project successfully simulates a chase game between two robots, showcasing the capabilities of ROS2 in handling complex robot interactions and real-time simulation. The implementation highlights the importance of precise robot modeling, efficient namespace management, and robust algorithm design in robotics simulations. The accurate implementation of TF (transform) frames was crucial for the project's success, as it enabled precise tracking and coordination between the robots, ensuring that Rick could effectively follow Morty's movements.

Key Insights
  • Robot Interaction: The project highlights the capability of ROS2 to manage complex interactions between multiple robots.
  • Simulation Accuracy: Accurate simulation of real-world robot behavior is achievable with proper modeling and algorithm implementation.
  • Algorithm Performance: The pursuit algorithm's effectiveness is critical for real-time robot tracking and following.
  • Namespace Management: Proper namespace management is essential to avoid conflicts and ensure independent control of multiple robots within the same environment.
