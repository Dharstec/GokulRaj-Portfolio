Introduction

The goal of this project is to develop a robust navigation system for the RB1 mobile robot, designed to operate within a warehouse environment. The system will enable the robot to autonomously create a map of the warehouse, localize itself within this map, and navigate to specified destinations using the Nav2 system. This document outlines the project's objectives, tools, techniques, development process, results, and key insights.

Objectives
  • Map Creation: Generate an accurate map of the warehouse environment.
  • Localization: Ensure the RB1 robot can accurately determine its position within the created map.
  • Autonomous Navigation: Enable the RB1 robot to autonomously navigate to specified goals within the warehouse using the Nav2 system.
  • System Integration: Integrate the mapping, localization, and navigation components into a cohesive system.
  • Testing and Validation: Validate the system's performance in both simulated and real-world environments.
Tools and Techniques:
  • ROS2 (Robot Operating System 2): Framework for developing robot software.
  • RViz2: Visualization tool for ROS.
  • Gazebo: Simulation environment for testing robot navigation.
  • Nav2 (Navigation 2): ROS2 package for robot navigation.
  • Simple Commander API: Interface for creating navigation routes.
  • Git: Version control system for managing project code.

Process and Development
1. Setting Up the Environment

First, a new Git repository named warehouse_project was created and cloned to ~/ros2_ws/src. The warehouse simulation was then launched using Gazebo to provide a controlled environment for development and testing.[Image]

×
2. Mapping

A package named cartographer_slam was created for the mapping task. The mapping process involved using the cartographer_node and occupancy_grid_node to generate a map of the warehouse, which was visualized using RViz2. The generated map was then saved in the config folder as warehouse_map_sim.[Image]

×
3. Localization

The localization_server package was created to handle the localization task. The amcl node was configured to enable the robot to localize itself within the warehouse map. The map_server node was used to publish the map, and localization was tested with the real robot to ensure accurate self-localization.[Image]

×
4. Navigation

The path_planner_server package was developed for the navigation task. This involved setting up nodes such as controller_server, planner_server, recoveries_server, bt_navigator, and RViz2 for the navigation process. Autonomous navigation was validated in both simulated and real environments by sending 2D navigation goals via RViz2, ensuring the robot could navigate to specified locations within the warehouse.[Image]

Results

The project successfully developed a navigation system for the RB1 mobile robot, capable of creating a map, localizing itself, and navigating within a warehouse environment. The system was tested and validated in both simulated and real-world scenarios, demonstrating reliable performance in autonomous navigation tasks.

Key Insights
  • Integration Complexity: Combining mapping, localization, and navigation components into a seamless system required careful integration and configuration.
  • Simulation vs. Real-World Testing: While the simulation provided a controlled environment for initial development, real-world testing was crucial for validating the system's robustness and handling real-world challenges.
  • Parameter Tuning: Fine-tuning parameters for the amcl and navigation nodes was essential to achieve accurate localization and smooth navigation.
  • Visualization Tools: RViz2 played a critical role in visualizing and debugging the robot's mapping, localization, and navigation processes.

This project lays the foundation for further enhancements in warehouse robotics, including optimizing navigation algorithms, integrating additional sensors for improved perception, and scaling the system for larger and more complex environments.
