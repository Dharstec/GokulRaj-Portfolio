Introduction

The goal of this project was to design and implement the kinematic model for the ROSBot XL robot in a simulated environment using Gazebo Sim (Ignition). The project involved testing the robot’s movement capabilities, developing a ROS2 package to publish wheel speeds, and transforming these into commands to effectively control the robot’s motion, including executing a complex figure-eight trajectory.

Objectives
  • To classify the ROSBot XL as holonomic or non-holonomic based on its movement capabilities.
  • To develop a kinematic model that accurately represents the robot’s real-world movements.
  • To implement a sequence of movements in simulation to demonstrate the robot’s capabilities, including a figure-eight trajectory.
Tools and Techniques:
  • Programming Languages: C++
  • Frameworks and Libraries: Eigen, ROS2, Gazebo Sim
  • Version Control: Git for version control

Process and Development

The project consisted of several critical phases, starting from the simulation setup to the execution of complex robot movements.

  1. Simulation Setup:
    • Created and configured the simulation environment using Gazebo Sim (Ignition).
  2. Kinematic Model Development and Movement Testing:
    • Movement Classification: Conducted tests to classify the ROSBot XL as either holonomic or non-holonomic, which is crucial for defining the appropriate kinematic model.
    • Wheel Velocities Publisher: Developed a C++ node that publishes the wheel speeds for each of the four wheels, enabling the robot to execute predefined movements like moving in different directions and rotating.
      • The node publishes messages of type Float32MultiArray on the /wheel_speed topic.
      • Sequence of movements: move forward, move backward, move sideways left, move sideways right, turn clockwise, turn counter-clockwise, and stop.
    • Kinematic Model Implementation: Created a node that subscribes to the /wheel_speed topic and transforms the wheel speed outputs into Twist messages to control the robot’s movements in the simulation accurately.(ViewMore...)
      • ×
      • This node publishes the Twist messages on the /cmd_vel topic to control the robot's motion based on the kinematic model.
  3. Advanced Trajectory Control:
    • Eight Trajectory Implementation: Developed a C++ package to command the robot to follow a figure-eight trajectory.
      • The node subscribes to the /odom topic to receive odometry information (position and orientation) of the robot and publishes wheel speed commands to the /wheel_speed topic.
      • The trajectory is defined by a series of waypoints, each representing a segment of the figure-eight path. The node calculates and publishes the necessary wheel speeds to follow these waypoints accurately.(ViewMore...)

Results

The project successfully implemented a kinematic model in ROS2, allowing the ROSBot XL to perform complex movements in a simulated environment. The robot demonstrated its ability to move forward, backward, sideways, rotate both clockwise and counter-clockwise, and precisely follow a figure-eight trajectory.

Key Insights
  • Developing complex trajectories like the figure-eight helped in assessing the robot’s performance and responsiveness to control commands.
  • The integration of a detailed kinematic model with ROS2 and Gazebo Sim facilitated a deeper understanding of robotic movement control in simulation environments.
