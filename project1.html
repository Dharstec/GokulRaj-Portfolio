Introduction

The project centres around the design, simulation, and control of the RB-1 Robotnik mobile robot using advanced robotics tools and frameworks. Leveraging the Unified Robot Description Format (URDF), Robotics Operating System (ROS), and Gazebo simulation environment, this project aims to develop a comprehensive understanding of robotic system modelling, simulation, and control mechanisms. By integrating these technologies, the project seeks to create a functional simulation of the RB-1 Robotnik capable of navigating and responding to commands within a virtual environment.

Objectives
  1. URDF Modeling: Develop a detailed URDF model of the RB-1 Robotnik, accurately representing its physical components and structure.
  2. Gazebo Simulation: Configure Gazebo to simulate the RB-1 Robotnik's mobility, integrating differential drive and sensor components for realistic navigation.
  3. ROS Control Implementation: Implement custom ROS services to enhance the robot's control capabilities, allowing for precise movement and interaction within the simulated environment.
Tools and Technologies
  • ROS (Robot Operating System): Framework for robotic software development.
  • Gazebo: Robotics simulation environment for testing and validation.
  • Linux operating system
  • RViz: 3D visualization tool for ROS.
  • Autodesk Fusion: CAD software for designing robot components.
  • Git: Version control for managing project iterations and collaboration.

Process and Development
Foundation: Simulating RB-1 Robotnik

The project began with defining the RB-1 Robotnik using URDF, ensuring each component and its physical characteristics were accurately represented. Detailed part schematics were designed using Autodesk Fusion, providing a precise model for simulation purposes.

Simulated Environment Setup in Gazebo

Transitioning to Gazebo, the simulated environment was tailored to support the RB-1 Robotnik's mobility. A differential drive mechanism was integrated, allowing for smooth and realistic navigation within the simulated terrain. This setup included configuring wheel dynamics and ensuring accurate interaction with the virtual ground.

Additionally, a Lidar sensor was incorporated into the URDF model to enhance the robot's navigation capabilities. Fine-tuning the sensor parameters within Gazebo enabled the robot to detect obstacles effectively and generate comprehensive environment maps, crucial for navigating autonomously in simulated scenarios.

Control Enhancement: Custom ROS Services

The project advanced to enhance the RB-1 Robotnik's control functionalities through custom ROS services. Within the my_rb1_ros package, rotate_service.cpp was developed to introduce a custom ROS service. This service empowered users to command precise rotational movements of the robot, enhancing its manoeuvrability and task execution capabilities within the simulated environment.

To facilitate seamless interaction and control, Rotate.srv, a custom ROS message structure, was implemented. This structure streamlined communication between users and the robot, enabling precise rotational commands and providing feedback upon task completion. Leveraging odometry data ensured accurate positional control and trajectory maintenance during operation.

Results

The culmination of the project resulted in a fully functional simulation of the RB-1 Robotnik mobile robot within the Gazebo environment. The robot successfully navigated and responded to control commands, demonstrating its capabilities in simulated scenarios. The integration of custom ROS services and precise sensor configurations contributed to the robot's robust performance and autonomy within the virtual world.
