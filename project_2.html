Introduction

This project is about the development of a user-friendly graphical interface (GUI) that enables remote interaction with these robots, accessible even to individuals with limited prior experience in robotics. Utilizing C++, ROS, and the CVUI library, this interface integrates seamlessly with ROS nodes, facilitating data exchange and service calls across the network. This project not only enhances user interaction but also reinforces C++ and ROS proficiency through the implementation of additional ROS nodes.

Objectives
  • To develop a user-friendly graphical interface (GUI) for remote interaction with a mobile robot using C++, ROS, and the CVUI library.
  • To integrate ROS nodes to facilitate seamless data exchange and service calls.
Tools and Techniques:
  • Programming Languages: C++
  • Frameworks and Libraries: ROS, CVUI, OpenCV
  • Simulation Environment: Gazebo
  • Version Control: Git

Process and Development

The project progresses through two main phases: backend ROS node development and GUI implementation.

ROS Nodes Development:

Initial Setup: Begin by cloning required repositories, setting up the ROS environment, and launching the simulation in Gazebo.[Image]

×

Robot Info Node: Develop a custom ROS node to publish essential robot details like description, serial number, IP address, and firmware version using a custom message format.

Distance Tracker Service: Create a ROS node that computes the robot's distance travelled using odometry data. This node offers a ROS service to provide distance information on request.

GUI Development: Utilize C++ and the CVUI library to build a dedicated ROS package for the GUI node. This node subscribes to the robot_info topic and interfaces with other ROS nodes to achieve the following functionalities:

  • Real-time display of updated robot information.
  • Teleoperation buttons for adjusting linear and angular velocities.
  • Visualization of the current robot position based on odometry data.
  • Integration of a button to trigger the /get_distance service and display the travelled distance.
  • Implementation of a button to reset the tracked distance using the /reset_distance service.[Video]

Results

The seamless integration of ROS nodes with the custom GUI has yielded a robust system that facilitates intuitive robot teleoperation and real-time status monitoring. This successful implementation underscores the efficacy of merging software development expertise with robotics, thereby enhancing user interaction through user-friendly interfaces tailored for complex robotic systems.
