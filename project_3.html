Introduction

The main objective of this project is to achieve a precise motion for the turtlebot3 in a given setup. The project encapsulates basic ROS2 services and actions to control a Turtlebot3 robot for autonomous patrolling and precise navigation to designated positions. The robot uses Laserscan to identify the nearest obstacle, which then decides and navigates to the safest calculated distance.

The goal is to enable the robot to patrol a designated area while avoiding obstacles, demonstrated in both a simulated environment and with a real Turtlebot3 robot.

Objectives
  • Develop patrol behaviour using ROS2 and C++ in a simulated environment.
  • Apply and test the developed program on a real Turtlebot3 robot in a citylab environment setup.
Tools and Techniques:
  • Programming Languages: C++
  • Frameworks and Libraries: ROS
  • Simulation Environment: Gazebo
  • Robot Hardware: Turtlebot3
  • Version Control: Git

Process and Development

The development process is segmented into initial simulation setup, programming the patrol behaviour, and real-world testing.

Simulation Setup

Initial Actions: The simulation environment in Gazebo is configured to mimic real-world scenarios, allowing the robot’s patrol behaviour to be tested and refined. This involves launching the simulation through ROS2 commands and ensuring the virtual Turtlebot3 is ready for programming.[Image]

Programming the Patrol Behavior

Creating the ROS2 Package: A package named robot_patrol was developed, containing a C++ file that defines the patrol behaviour. This includes subscribing to laser sensor topics to navigate and avoid obstacles based on the data received.

Algorithm Implementation: A simple algorithm analyses 180 degrees in front of the robot, determining the safest direction by identifying the largest distance free of obstacles. This direction guides the robot’s movement, ensuring continuous patrolling without collisions.
Furthermore, Services and action goals were implemented to enhance the complexity for more precision movements when patrolling.[Image]

×
Services for Robot Navigation

Creating the Service: Before creating the service a custom service interface GetDirection.srv was created to take laser_data as request and direction as response. A ROS2 service server named direction_service.cpp was developed in the same package. This service receives laser data from the Turtlebot3 and analyses it to determine the safest direction for the robot to move, enhancing its obstacle avoidance capabilities.

Integration with Patrol Behavior

Modifying the Patrol Script: The existing patrol behaviour script, patrol_with_service.cpp, was modified to include calls to the newly developed ROS2 service. This integration allows for dynamic obstacle avoidance based on real-time sensor data. After successfully testing the navigation node in simulation, it was deployed to the real robot.

Action Server Creation

Developing the Action Server: An action server named go_to_pose_action.cpp was created. This server accepts goal positions and navigates the Turtlebot3 to these locations using odometry and velocity commands, showcasing advanced robot control techniques.[Image]

Real-world Testing

Deployment on the Real Robot: Before real-world deployment, the patrol behaviour was rigorously tested in the simulation to ensure reliable obstacle avoidance. After multiple successful simulation tests, the program was executed on an actual Turtlebot3 robot. This real-world test was conducted remotely, with the robot located in Barcelona, Spain, with the assistance of The Construct organization. Adjustments and debugging were carried out in real-time to adapt the patrol behavior to the nuances of the physical environment.[Video 1], [Video 2]

Results

The project achieved its goal of developing an autonomous patrol behaviour that effectively navigates and avoids obstacles, both in a simulated environment and in real-world testing. The seamless transition from simulation to real-world application underscores the robustness of the developed program and the adaptability of ROS2 for robot programming.

Key Insights
  • The importance of simulation in robotics development was highlighted, serving as a critical step before real-world application.
  • The adaptability of ROS2 in integrating with both simulated and real environments was evident, showcasing its potential for developing complex robotic behaviours with minor tweaks in the code.
  • Future enhancements could include refining the obstacle avoidance algorithm for more complex environments and improving the robustness of the patrol behaviour.
Future Enhancements
  • Algorithm Improvement: Enhance the obstacle avoidance algorithm to handle more complex and dynamic environments.
  • Sensor Integration: Incorporate additional sensors for better perception and decision-making.
  • Advanced Behaviours: Develop more advanced patrol patterns and behaviours for specific applications.

This project not only demonstrates the foundational capabilities of ROS2 but also paves the way for more sophisticated autonomous robotic applications in the future.
